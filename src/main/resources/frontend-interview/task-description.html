Frontend Developer Task - File Manager

Introduction

Welcome to our frontend developer task. We believe in assessing your coding skills by giving you the opportunity to showcase your abilities through practical application. This task aims to understand your coding style and will serve as a foundation for further discussions during the interview. Remember, there's no right or wrong solution; we're interested in your approach to problem-solving.

Objective

The objective of this task is to create a frontend application that seamlessly integrates with the provided backend API to enable file uploads and management.

Requirements

  • The application shows available images
  • Each image must be represented by a thumbnail, as well as it's name and size
  • The application shall include a file upload triggered by a button (standard behavior)
  • Multiple file upload shall be possible
  • File upload progress is shown in some way (numbers, percentage, progress bar, ...)
  • File upload status is visible (error or success)
  • Provide the source code of the frontend application along with any necessary instructions for running the application locally.

Bonus (extra credit)

  • Drag-and-drop file upload
  • Test coverage >80%
  • Responsive design
  • Pagination (endpoint /api/v1/filepage/{number})
  • Include any additional documentation or notes relevant to the implementation

Constraints

  • Use Angular 14+ for frontend development.
  • You are free to choose any styling library/framework of your choice for UI design (e.g., Bootstrap, Material Design).
  • Ensure compatibility with modern web browsers.

Examples

Getting data about files

To get a list of files, you can use one the following endpoints

  • /api/v1/files - gets a list of all files
  • /api/v1/files/{id} - gets a list of all articles separated in pages

To get a thumbnail of a desired size, you can use the parameter size on the /api/v1/files/{id}. For example size=100 will return a thumbnail of max size 100x100px.

Uploading a file

To upload a file, you can use the following endpoint:

  • /api/v1/files - POST request as multipart file

Additional information

Please note that the REST endpoint enables only a single file upload - you will have to implement the multiple file upload with angular!

All files are supported, but thumbnails are generated only for JPEG (JPG), PNG and GIF

Max upload file size is: 5MB
