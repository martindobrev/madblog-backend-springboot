Welcome to our Frontend Developer Task

Why a programming task?

In order to get an impression of your coding skills we are asking you to do what you do best - write some code. Solving the following task will give us a basic understanding of your style and will be used as a basis for further questions and open technical discussion. There is no right or wrong solution, we just want to see how you will approach the problem.

The Task

Create a simple web application with the following pages:

  • Home - displays a menu with all available pages. Also displays a list of blog articles. Basically it is an entry point for the blog itself.
  • Webpage - when clicking on the menu with the available pages, the selected page shall be loaded.
  • Blog details page - Articles on the main page shall not be displayed completely. Clicking the title of each article will display the full content on a separate page

Instructions

Getting data about articles

To get a list of articles, you can use one of the endpoints of the article controller:

  • /api/v1/articles - gets a list of all articles
  • /api/v1/articles/page/{pageNumber} - gets a list of all articles separated in pages

To get a single article, you can use the endpoint /api/v1/articles/{articleId}

Note that when you get a list of articles, the htmlContent field will be empty. HTML will be rendered only when getting a single article.
Getting available web pages
  • /api/v1/menu - get a list of available pages for the menu.
  • /api/v1/page/{slug} - get a single page by it's slug (A URL slug is the part of the URL after the last backslash)
Additional information

The API is based on an old tutorial project, so there are some additional controllers and endpoints you won't need. You can simply ignore them.

All articles and pages contain some images. They are controlled by the FileController endpoints. You will probably not directly need them, they will usually work out-of-the-box.
